# wrangler integration tests

Verifies that `rosie-skills` keeps working as the library
[wrangler](https://github.com/cloudflare/workers-sdk/tree/main/packages/wrangler)
consumes. Wrangler uses rosie as an **external (unbundled)** dependency: on
`wrangler setup --install-skills` it dynamically imports `rosie-skills` and
calls `rosie.agents()` then `rosie.install("cloudflare/skills", { global: true,
agent, lockfile: false })`. These tests guard that exact seam.

## Levels

- **Level 1 — contract** (`contract.mjs`): hermetic. Imports the freshly built
  package by **bare specifier** (validating the `exports` map / ESM resolution,
  not a deep path into `dist/`) and replicates wrangler's glue code. No wrangler
  install, fully offline.
- **Level 2 — e2e**: installs the real `wrangler` from npm with `rosie-skills`
  overridden to this checkout's build (`overrides: { "rosie-skills": "file:…" }`),
  then runs the real `wrangler setup --install-skills` binary and asserts the
  skills land on disk. Needs npm-registry access.

Both point rosie at the regression suite's mock GitHub server
(`tests/regression/lib/mock_server.py`) via `ROSIE_GITHUB_BASE_URL`, serving a
`cloudflare/skills` fixture under `tests/regression/fixtures/sources/cloudflare/`.
No real github.com traffic.

## Running

```sh
./run.sh                       # both levels
./run.sh --no-e2e              # contract only (fully offline)
./run.sh --wrangler-version X  # pin wrangler (default: latest)
./run.sh --port 8788           # mock server port
./run.sh --keep-tmp            # keep scratch dirs for inspection
```

Env equivalents: `SKIP_E2E=1`, `WRANGLER_VERSION=X`, `PORT=N`, `KEEP_TMP=1`.

## Notes

- The skills install runs in wrangler's command wrapper, *before* the `setup`
  body, so the e2e level asserts on installed files and tolerates `setup`'s own
  exit code (autoconfig fails in an empty project, which is expected).
- Level 2 confirms wrangler resolves *this* build by checking the installed
  `rosie-skills` version sentinel (`0.0.0`; the published package is `^0.7.x`)
  and that `overrides` left no nested copy under `wrangler`.
- `cloudflare/skills` tarball fixtures are generated by
  `tests/regression/fixtures/build.sh` (auto-run by `run.sh` when missing).
